# 個人日記Burn - 自分で浄化タイミングを選ぶ

## コンセプト

> **「手放す準備ができたら、自分で浄化できる」**

ネガティブを書き出すことは治療的ですが、**削除のタイミングは本人が決めるべき**です。

---

## 焚き火広場 vs 個人日記Burn

| 特徴 | 🔥 焚き火広場 | 📔 個人日記Burn |
|---|---|---|
| **閲覧** | コミュニティ（匿名） | **オーナーのみ** |
| **削除** | 5分で自動 | **ユーザーが選択** |
| **目的** | 即座の吐き出し・共感 | 繰り返し読み返し・客観視 |
| **記録** | 統計のみ | 統計のみ |

---

## 心理学的根拠

### Expressive Writing療法の最適化

James Pennebakerの研究：
- 1回15-20分、3-5日間続けることが推奨
- 即座に削除するより、**数日後に読み返す**方が効果的な場合も

### 自己決定理論

心理学における**Self-Determination Theory**：
- 自分で選択・コントロールできることが重要
- 強制的な削除より、**自分で手放すタイミングを選ぶ**方が治療的

### 儀式化の効果

意図的に「浄化する」という行為を選択することで：
- より深いカタルシスを得られる
- 感情との向き合いに区切りをつけられる
- セルフケアの実感が強まる

## ユースケース

### パターン1: 即座の吐き出し

```
夜、辛いことがあった
→ 個人Burn日記に書く
→ 書き終わった瞬間に「浄化する」
→ すっきりして眠る
```

### パターン2: 繰り返し読み返し

```
1日目: 怒りを書き出す
2日目: 読み返す → まだ怒っている
3日目: 読み返す → 少し客観視できた
4日目: 読み返す → もう大丈夫
→ 「浄化する」ボタンを押す
```

### パターン3: パターン認識

```
週に2-3回、似たような内容を書く
→ 「あれ、同じパターンだ」と気づく
→ カルテに正式に記録する
→ 個人Burn日記は浄化
```

## データ設計

### PersonalBurnDiary

```typescript
interface PersonalBurnDiary {
  id: string;
  userId: string; // オーナー
  content: string;
  createdAt: Timestamp;
  
  // 感情データ
  category: 'anger' | 'sadness' | 'anxiety' | 'frustration';
  emotionIntensity: 1 | 2 | 3 | 4 | 5;
  
  // 浄化関連
  isBurned: boolean;
  burnedAt?: Timestamp;
  burnedByUser: boolean; // ユーザーが手動で浄化
  readCount: number; // 読み返した回数
  
  // セーフティネット
  autoExpiresAt: Timestamp; // 30日後
  
  // プライバシー（常に固定）
  isPrivate: true;
  viewableBy: [userId];
  
  // AI分析
  mochisuraComment?: string;
}
```

## プライバシー保護

### Firestore Rules

```javascript
match /personalBurnDiaries/{diaryId} {
  // 本人のみアクセス可能
  allow read, write: if request.auth.uid == resource.data.userId;
  
  // 作成時のバリデーション
  allow create: if request.auth.uid == request.resource.data.userId
                && request.resource.data.isPrivate == true;
}
```

## まとめ

### 個人Burn日記の価値

1. **自己決定感** - 手放すタイミングを自分で選べる
2. **客観視の機会** - 数日後に読み返して成長を実感
3. **完全なプライバシー** - 誰にも見られない安心感
4. **儀式化された浄化** - 能動的に「手放す」体験

**ユーザーは自分のペースでネガティブと向き合い、準備ができたら手放すことができます。🌸**
